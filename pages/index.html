Intent, Custody, and Proof:
Toward Accountable Execution in Agentic Software Engineering

Alex H. Raber
Decapod Labs
Download Paper PDF View GitHub Repository

Abstract

Large language model (LLM) agents are transitioning from conversational programming assistants to autonomous executors capable of editing codebases, running build tools, executing tests, and managing multi-file software deployments. Despite this shift, current agent workflows still rely on standard prompt transcripts and code diffs as the primary logs of execution. As agent runs become concurrent, long-running, and tool-intensive, this transcript-only paradigm exposes repositories to significant failure modes: lost intent, authority leakage, concurrent state corruption, and unverifiable completion claims.


This paper frames agentic software engineering as a distributed execution problem. We introduce a framework built around four primitives to achieve accountable execution: Intent (durable target objectives), Custody (isolated environment bounds and tool permissions), Trajectory (immutable, kernel-logged audit trails), and Proof (cryptographically anchored, machine-checkable evidence of completion). We present Decapod, a local-first governance kernel, as a reference implementation of this model. We evaluate governed execution across 35 multi-step software engineering tasks against prompt-only and checklist-disciplined baselines. Our findings show that explicit custody and verification gates significantly reduce false completion claims, prevent concurrent edit conflicts, and decrease the human developer effort required to audit and merge autonomous code contributions.

Core Primitives

I

Intent

A durable, structured task manifest tracked outside the agent's context window. Ensures instructions and success criteria survive process interruptions, rate limits, and context drift.

C

Custody

Isolated execution workspaces (git worktrees) and restricted tool command scopes. Prevents authority leakage, unauthorized credential usage, and concurrent file collisions.

T

Trajectory

An append-only, kernel-logged audit trail of all commands, file diffs, and intermediate validation checks. Recorded by the governance kernel to ensure safety and allow full state recoverability.

P

Proof

Cryptographically signed, machine-checkable evidence containing successful test runs, compiler output logs, and workspace tree hashes. Shifts completion verification from conversational assertions to checkable claims.
