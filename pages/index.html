Intent, Custody, Trajectory, and Proof:
Toward Accountable Execution in Agentic Software Engineering

Alex H. Raber
Decapod Labs
Download Paper PDF View Repository

Abstract

Large language model (LLM) agents are transitioning from conversational programming assistants to autonomous executors capable of editing codebases, running build tools, executing tests, and managing multi-file software deployments. Despite this shift, current agent workflows still rely on standard prompt transcripts and code diffs as the primary logs of execution. As agent runs become concurrent, long-running, and tool-intensive, this transcript-only paradigm exposes repositories to significant failure modes: lost intent, authority leakage, concurrent state corruption, and unverifiable completion claims.


This paper frames agentic software engineering as a distributed execution problem. We introduce a framework built around four primitives to achieve accountable execution: Intent (durable target objectives), Custody (isolated environment bounds and tool permissions), Trajectory (event-backed records of governance activity), and Proof (machine-checkable validation and provenance evidence). We present Decapod, a local-first governance kernel, as a reference implementation of this model. We propose an empirical evaluation protocol to measure governed execution across 35 multi-step software engineering tasks against prompt-only and checklist-disciplined baselines. We discuss how explicit custody and verification gates are projected to reduce false completion claims, prevent concurrent edit conflicts, and decrease the human developer effort required to audit and merge autonomous code contributions.

Core Primitives

Intent

Durable Intent

A durable task contract represented across plans, todos, specs, unknowns, stop conditions, and proof expectations.

Custody

Isolated Custody

Isolated workspaces (git worktrees) and execution sandboxes. Reduces workspace collision and unsafe repository mutation risk; credential isolation depends on the configured runtime boundary.

Trajectory

Auditable Trajectory

Event-backed governance records for task, broker, proof, and validation activity. Designed to anchor execution history and support recoverability.

Proof

Verifiable Proof

Machine-checkable validation, proof-run, workunit, and provenance evidence linked to governed task state.
